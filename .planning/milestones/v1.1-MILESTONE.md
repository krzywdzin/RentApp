# Milestone v1.1: Quality, Polish & UX Improvements

**Completed:** 2026-03-25
**Duration:** Single session
**Phases:** 5 (10-14)
**Plans:** 14 total
**Requirements:** 29/29 complete

## Summary

v1.1 focused on quality, polish, and developer experience improvements across the entire RentApp monorepo. All work was non-functional — no new user-facing features, only improvements to existing ones.

## What Shipped

### Phase 10: Mobile UX Polish (3 plans)
- Loading skeletons on all list screens (vehicle selection, return wizard)
- Customer search hint text ("type 2+ chars") and fetch spinner
- Error state with retry on rental detail screen
- Return wizard rentalId guards (prevent 0 km mileage bug)
- OfflineBanner in return wizard layout
- Polish status labels replacing raw enum values
- Dashboard greeting email fallback, PDF error toast

### Phase 11: Web Admin Panel Polish (3 plans)
- Complete user management page (list, edit, deactivate, password reset)
- Rental list shows vehicle registration + customer names (not UUIDs)
- Contract tab wired with real data on rental detail
- Edit rental form with Zod validation + inline errors
- Audit date/actor filters wired to API
- Error states on dashboard and contract list
- Login page using shadcn/ui design system

### Phase 12: TypeScript Strictness (3 plans)
- ~60 `any` types removed from backend services
- Typed DTOs for rentals, contracts, damage, photos, vehicles services
- PortalRequest interface for portal controller
- Typed mutation hooks in web admin
- PortalReturnInspectionData replacing `returnData: any`

### Phase 13: Dependencies & Performance (2 plans)
- react-native-webview as explicit dependency
- Expo SDK 54 version alignment (tilde ranges)
- Contract list N+1 query eliminated (single API call)
- Server-side rental filtering for customer/vehicle detail pages

### Phase 14: Test Coverage (3 plans)
- Vitest + RTL infrastructure for web admin (12 component tests)
- jest-expo smoke tests for mobile (12 tests)
- API coverage threshold enforcement (35% minimum)

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Skip research for all phases | Polish/quality work on known codebase — no new domain |
| 35% API coverage threshold | Baseline was 40% — regression floor, not aspirational target |
| Vitest for web (not Jest) | Better ESM support, faster, native TypeScript |
| Polish-only error messages | App is Polish-only per v1 scope |
