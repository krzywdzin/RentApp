    **Write unit tests FIRST (TDD red)** for processReturn, extend, and rollback. Replace remaining `it.todo()` stubs in `rentals.service.spec.ts`.

    **Implement processReturn(rentalId, dto, actorId):**
    1. Find rental with vehicle include (throw NotFoundException if missing)
    2. Call `validateTransition(rental.status, RentalStatus.RETURNED)` — only ACTIVE and EXTENDED can return
    3. Use `prisma.$transaction()` to:
       a. Update rental: `status=RETURNED`, `returnMileage=dto.returnMileage`, `returnData=dto.returnData ?? Prisma.JsonNull`, `notes` appended if provided
       b. Update vehicle: `status=AVAILABLE`
    4. Re-fetch rental with includes to get full data
    5. Emit `'rental.returned'` event with `{ rentalId, vehicleId, customerId, returnedBy: actorId }`
    6. Return rental (which includes both `handoverData` and `returnData` for side-by-side comparison)
    7. Attach `__audit` metadata: old status -> RETURNED

    **Implement extend(rentalId, dto, actorId):**
    1. Find rental (throw NotFoundException if missing)
    2. Validate current status allows extension: call `validateTransition(rental.status, RentalStatus.EXTENDED)`
    3. Parse newEndDate, validate it's after current startDate
    4. Check for overlaps with new date range: `checkOverlap(rental.vehicleId, rental.startDate, newEndDate, rentalId)` — if conflicts exist, throw ConflictException (extensions don't have override, unlike creation)
    5. Calculate new days from startDate to newEndDate
    6. If `dto.totalPriceNet` provided (admin override): use it directly
    7. Otherwise: recalculate using `calculatePricing({ dailyRateNet: rental.dailyRateNet, days: newDays })`
    8. Use `prisma.$transaction()` to update rental: `status=EXTENDED`, `endDate=newEndDate`, pricing fields updated
    9. Emit `'rental.extended'` event with `{ rentalId, customerId: rental.customerId, newEndDate, extendedBy: actorId }`
    10. Return updated rental with `__audit` metadata

    **Implement rollback(rentalId, dto, actorId):**
    1. Find rental (throw NotFoundException if missing)
    2. Call `validateTransition(rental.status, dto.targetStatus, true)` — uses admin rollback map
    3. Determine vehicle status side effect:
       - Rolling back to ACTIVE or EXTENDED => vehicle should be RENTED
       - Rolling back to DRAFT => vehicle should be AVAILABLE
    4. Use `prisma.$transaction()` to:
       a. Update rental: `status=dto.targetStatus`
       b. If rolling back from RETURNED: clear returnMileage and returnData
       c. Update vehicle status accordingly
    5. Emit `'rental.rolledBack'` event with `{ rentalId, from: oldStatus, to: dto.targetStatus, reason: dto.reason, rolledBackBy: actorId }`
    6. Return updated rental with `__audit` metadata including reason in changes
