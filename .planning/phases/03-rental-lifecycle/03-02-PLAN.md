    **Implement RentalsService methods:**

    **create(dto, actorId):**
    1. Validate vehicle exists and is not RETIRED
    2. Validate customer exists
    3. Calculate days: `Math.ceil((endDate - startDate) / (1000 * 60 * 60 * 24))` or minimum 1 day
    4. Call `calculatePricing({ totalPriceNet: dto.totalPriceNet, dailyRateNet: dto.dailyRateNet, days })`
    5. Call `checkOverlap(dto.vehicleId, startDate, endDate)`
    6. If conflicts exist and `dto.overrideConflict !== true`: return `{ rental: null, conflicts }` (HTTP 409 Conflict from controller)
    7. Use `prisma.$transaction()` to:
       a. Create rental with pricing fields, `overrodeConflict: conflicts.length > 0`
       b. If `dto.status === 'ACTIVE'`: update vehicle status to RENTED, store handoverData
    8. Emit `'rental.created'` event
    9. Return the created rental with vehicle and customer includes

    **checkOverlap(vehicleId, startDate, endDate, excludeRentalId?):**
    Use `prisma.$queryRaw` with tstzrange overlap operator:
    ```sql
    SELECT id, start_date as "startDate", end_date as "endDate", status
    FROM rentals
    WHERE vehicle_id = $1
      AND status NOT IN ('RETURNED')
      AND tstzrange(start_date, end_date, '[)') && tstzrange($2::timestamptz, $3::timestamptz, '[)')
      AND ($4::text IS NULL OR id != $4)
    ```
    Use Prisma.sql tagged template for parameterized query. Return array of conflicting rental summaries.

    **activate(rentalId, dto, actorId):**
    1. Find rental (throw NotFoundException if missing)
    2. Call `validateTransition(rental.status, RentalStatus.ACTIVE)`
    3. Use `prisma.$transaction()` to:
       a. Update rental: status=ACTIVE, handoverData=dto.handoverData
       b. Update vehicle: status=RENTED
    4. Emit `'rental.activated'` event
    5. Return updated rental

    **findAll(filters?):**
    Return rentals with vehicle and customer relations. Support optional status filter. Order by startDate desc.

    **findOne(id):**
    Find rental by id with vehicle and customer includes. Throw NotFoundException if not found.

    **getCalendar(from, to):**
    1. Find all vehicles (non-archived)
    2. Find all rentals in the date range (where startDate <= to AND endDate >= from, excluding RETURNED unless in range)
    3. Group rentals by vehicleId
    4. For each vehicle, check if any rentals overlap each other (hasConflict flag)
    5. Return `CalendarResponse` format: vehicles array with registration, make, model, and their rentals

    All service methods that mutate return `{ ...rental, __audit: { action, oldValues, newValues } }` for the audit interceptor.
