<objective>
Persist wizard state across backgrounding and fix navigation guards so no data is lost when the app is backgrounded mid-wizard or the user presses back on intermediate steps.

Purpose: The signatures flow stores rentalId, contractId, and currentIndex in local useState -- if the app backgrounds, these are lost and a duplicate rental is created on resume. Navigation guards only fire on the first wizard screen, letting users lose data on intermediate steps. Return submission uses dismissAll which lands on unpredictable screens.

Output: Reliable state persistence in Zustand stores, discard dialogs on all wizard steps, correct post-submission navigation.
</objective>
