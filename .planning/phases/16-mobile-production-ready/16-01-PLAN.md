# Plan 16-01: Dynamic API URL, Keyboard Handling & EAS Build Config

**Phase:** 16 — Mobile Production Ready
**Requirements:** MOBP-01, MOBP-02, MOBP-03, MOBP-05

## Tasks

### Wave 1: Core Fixes (parallel)

**Task 1.1: Dynamic API URL from config**
- In `apps/mobile/src/api/client.ts`, replace hardcoded `http://192.168.0.26:3000` with:
  ```typescript
  import Constants from 'expo-constants';
  const API_URL = Constants.expoConfig?.extra?.apiUrl ?? 'http://localhost:3000';
  ```
- Remove all hardcoded IP references
- Verify app.config.ts already has `extra.apiUrl` from EXPO_PUBLIC_API_URL env var

**Task 1.2: Keyboard-aware forms**
- In contract.tsx (contract review screen), wrap content in KeyboardAvoidingView + ScrollView
- Ensure dates.tsx form handles keyboard properly
- Use `behavior="padding"` on iOS, `behavior="height"` on Android

**Task 1.3: Vehicle selection FlatList fix**
- In vehicle.tsx, check for VirtualizedList nesting issues
- If FlatList is nested inside ScrollView, refactor to use FlatList as the outer container with ListHeaderComponent

### Wave 2: EAS Build Config

**Task 2.1: EAS Build profiles**
- Update `apps/mobile/eas.json` with proper profiles:
  - development: developmentClient true, distribution internal
  - preview: android buildType apk, distribution internal
  - production: android buildType app-bundle, autoSubmit false
- Ensure app.config.ts has correct EAS project configuration
