# Plan 16-02: Photo Walkthrough Step in Rental Wizard

**Phase:** 16 — Mobile Production Ready
**Requirements:** MOBP-04

## Tasks

### Wave 1: Photo Capture Screen

**Task 1.1: Create photo walkthrough screen**
- Create `apps/mobile/app/(tabs)/new-rental/photos.tsx`
- Implement a multi-photo capture screen with positions: front, rear, left_side, right_side
- Use expo-image-picker (launchCameraAsync) for photo capture — simpler than expo-camera
- Display photo grid with position labels and capture/retake buttons
- Store photo URIs in rental draft store
- Add "Skip photos" option (photos are recommended but not blocking)
- Navigation: contract → photos → signatures

**Task 1.2: Update wizard flow**
- Add 'photos' screen to the Stack in `_layout.tsx`
- Update WIZARD_LABELS to include "Zdjecia" step
- In contract.tsx, change next navigation from signatures to photos
- In photos.tsx, navigate to signatures on continue
- Update rental draft store to include photoUris field

**Task 1.3: Upload photos on rental creation**
- In signatures.tsx (where rental creation happens), after rental/contract creation:
  - Create a walkthrough via POST /walkthroughs
  - Upload each photo via POST /walkthroughs/{id}/photos
  - Submit walkthrough via POST /walkthroughs/{id}/submit
- Make upload non-blocking — if upload fails, show warning but don't block rental

## Success Criteria
- [ ] Photo screen appears between contract review and signatures
- [ ] Employee can capture front/rear/left/right photos
- [ ] Photos upload to API after rental creation
- [ ] Skip option available for time-sensitive situations
