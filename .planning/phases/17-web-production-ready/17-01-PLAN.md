# Plan 17-01: Token Refresh, Form Validation, Error States & Build Fix

**Phase:** 17 — Web Production Ready
**Requirements:** WEBP-01, WEBP-02, WEBP-03, WEBP-04

## Tasks

### Wave 1: Token Refresh & Build Fix (parallel)

**Task 1.1: Token refresh on 401**
- In `apps/web/src/lib/api-client.ts`, add token refresh logic:
  - On 401 response, call POST /api/auth/refresh with stored refresh token
  - Queue concurrent requests while refresh is in progress
  - Retry original request with new token
  - On refresh failure, redirect to /login
- Store tokens in httpOnly cookies via BFF proxy (if already implemented) or localStorage

**Task 1.2: Fix build errors**
- Fix prettier formatting in wynajmy/nowy/page.tsx, test files
- Fix unused variable onStatusChange in wynajmy/nowy/page.tsx
- Fix requires import in test/setup.ts
- Run `pnpm --filter web build` to verify zero errors

### Wave 2: Form Validation & Error States

**Task 2.1: Consistent form validation**
- Verify create vehicle, edit vehicle, create rental, edit rental forms all use Zod + react-hook-form
- Add missing inline field-level error display where absent
- Ensure error messages are in Polish

**Task 2.2: Error states with retry on data pages**
- Add error state rendering to dashboard, rental list, customer list, vehicle list pages
- Pattern: if query isError, show error card with message and retry button (refetch)
- Use consistent error UI component (shadcn Alert or Card with destructive variant)

## Success Criteria
- [ ] 401 responses trigger automatic token refresh
- [ ] `pnpm --filter web build` succeeds with zero errors
- [ ] All forms show inline validation errors
- [ ] Data pages show error states with retry buttons
