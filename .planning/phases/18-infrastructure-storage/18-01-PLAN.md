# Plan 18-01: Dockerfile, Health Check, R2 Storage & Deployment Config

**Phase:** 18 — Infrastructure & Storage
**Requirements:** INFRA-01, INFRA-02, INFRA-03, INFRA-04, INFRA-05

## Tasks

### Wave 1: Core Infrastructure (parallel)

**Task 1.1: API Dockerfile**
- Create `apps/api/Dockerfile` — multi-stage build:
  - Stage 1 (deps): node:20-alpine, copy package.json + pnpm-lock, install deps
  - Stage 2 (build): copy source, run `pnpm build`, generate prisma client
  - Stage 3 (production): node:20-alpine, copy dist + node_modules + prisma, non-root user, expose port
- Create `apps/api/.dockerignore`
- Dockerfile must work from repo root context (monorepo — needs workspace files)

**Task 1.2: Enhanced health check**
- Replace simple HealthController in app.module.ts with a proper one that checks:
  - DB: PrismaService.$queryRaw`SELECT 1`
  - Redis: check connection via ConfigService REDIS_URL
  - Storage: check s3Available flag from StorageService
- Return: `{ status: "ok"|"degraded", db: true/false, redis: true/false, storage: true/false, timestamp }`
- Inject PrismaService and StorageService into health controller
- Move HealthController to its own file: `apps/api/src/health/health.controller.ts`

**Task 1.3: Web deployment config**
- In `apps/web/next.config.ts`, add `output: 'standalone'` for Docker/Railway deployment
- Create `apps/web/Dockerfile` — multi-stage (deps, build, production with standalone output)

### Wave 2: Railway & Deployment Config

**Task 2.1: Railway configuration**
- Create `Procfile` at repo root for Railway:
  ```
  web: node apps/api/dist/main.js
  ```
- Or use railway.toml with health check path and build commands
- Add `apps/api/railway.toml` with:
  ```toml
  [build]
  builder = "dockerfile"
  dockerfilePath = "apps/api/Dockerfile"

  [deploy]
  healthcheckPath = "/health"
  healthcheckTimeout = 30
  restartPolicyType = "on_failure"
  ```

**Task 2.2: Verify R2 compatibility**
- StorageService already uses @aws-sdk/client-s3 with forcePathStyle
- R2 works with S3 SDK — just needs correct endpoint and credentials
- Document in .env.example: S3_ENDPOINT for R2 is `https://<account-id>.r2.cloudflarestorage.com`
- No code changes needed — just env config

## Success Criteria
- [ ] Dockerfile builds successfully
- [ ] Health check returns DB/Redis/Storage status
- [ ] Next.js configured for standalone deployment
- [ ] Railway config files present
