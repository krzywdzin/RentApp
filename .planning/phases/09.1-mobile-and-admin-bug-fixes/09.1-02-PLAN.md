The react-native-signature-canvas is a WebView-based component. When there are elements above the canvas (header 48px, instruction text ~37px, margins), the WebView's internal coordinate system doesn't match the touch coordinates because the WebView occupies only part of the screen.

Fix approach:

1. Add `androidHardwareAccelerationDisabled={true}` prop to `SignatureCanvas` -- this fixes a known Android coordinate offset issue with WebView.

2. Update the `SIGNATURE_WEB_STYLE` to add explicit CSS that fixes the canvas sizing:
```
body, html { margin: 0; padding: 0; width: 100%; height: 100%; overflow: hidden; }
canvas { width: 100% !important; height: 100% !important; }
.m-signature-pad { position: absolute; top: 0; left: 0; right: 0; bottom: 0; }
.m-signature-pad--body { position: absolute; top: 0; left: 0; right: 0; bottom: 0; border: none; }
.m-signature-pad--footer { display: none; }
```

3. Ensure the canvas View wrapper has `position: 'relative'` in styles and the SignatureCanvas fills it completely.

4. Add `trimWhitespace={false}` to prevent automatic cropping that could affect coordinate mapping.

5. Add `style={{ flex: 1 }}` directly to the SignatureCanvas component to ensure it fills the canvas container.

  <action>
The react-native-signature-canvas onOK callback returns a data URI string like `data:image/png;base64,iVBOR...`. The shared Zod schema validates this with a regex that accepts both `data:image/png;base64,` prefix and raw `iVBOR` prefix. However, in `contracts.service.ts` line 222:

```typescript
const signatureBuffer = Buffer.from(dto.signatureBase64, 'base64');
```

This creates a corrupt buffer when the input has the `data:image/png;base64,` prefix because `Buffer.from` tries to decode the entire string including the prefix as base64.

Fix in `apps/api/src/contracts/contracts.service.ts`, in the `sign` method, before the `Buffer.from` call:

```typescript
// Strip data URI prefix if present
const rawBase64 = dto.signatureBase64.replace(/^data:image\/\w+;base64,/, '');
const signatureBuffer = Buffer.from(rawBase64, 'base64');
```

Also apply the same fix for the `damageSketchBase64` in the `create` method (line 163):

```typescript
const rawSketchBase64 = dto.damageSketchBase64.replace(/^data:image\/\w+;base64,/, '');
const buffer = Buffer.from(rawSketchBase64, 'base64');
```

This ensures all base64 image data is properly decoded regardless of whether the client sends a data URI or raw base64.
