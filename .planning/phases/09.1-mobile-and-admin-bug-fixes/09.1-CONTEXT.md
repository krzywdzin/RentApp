<decisions>
## Bug Reports

### BUG-1: Signature canvas draws in wrong position
- **Where:** Mobile app, signature capture screen (`apps/mobile/src/components/SignatureScreen.tsx`)
- **Symptom:** Drawing with finger/stylus renders in a different position than where user touches. Sometimes draws "something else completely."
- **Likely cause:** Canvas coordinate offset issue — WebView-based react-native-signature-canvas doesn't account for screen offset/SafeArea in landscape mode.

### BUG-2: Rental creation fails after signing
- **Where:** Mobile app, rental wizard step 5 → API
- **Symptom:** "Nie udalo sie utworzyc najmu" error after completing signature flow
- **Likely cause:** API returns error during rental creation or contract creation. Need to check API response and mobile error handling.

### BUG-3: Cannot delete draft rentals
- **Where:** Mobile app or admin panel, rental list/detail
- **Symptom:** User cannot remove rentals that are in DRAFT status
- **Likely cause:** No delete endpoint for DRAFT rentals, or missing UI button.

### BUG-4: Admin cannot create new user accounts
- **Where:** Admin panel (`apps/web`), user management
- **Symptom:** Cannot create new employee logins from admin panel
- **Likely cause:** No user creation page in admin panel (Phase 5 focused on entity CRUD, may have missed user management UI).

### BUG-5: Cannot search existing customers in mobile app
- **Where:** Mobile app, rental wizard step 1 (customer search)
- **Symptom:** Search returns no results for already-added customers
- **Likely cause:** API search endpoint issue, or mobile app sends wrong query format.

### BUG-6: Mobile app is slow and laggy
- **Where:** Mobile app, general
- **Symptom:** App feels slow, UI lags
- **Likely cause:** Missing React.memo, unnecessary re-renders, heavy list rendering without virtualization, or debug mode overhead.

### BUG-7: Mobile app crashes randomly
- **Where:** Mobile app, random screens
- **Symptom:** App quits unexpectedly
- **Likely cause:** Unhandled promise rejections, missing error boundaries, or memory issues from large images.

### Claude's Discretion
- Exact fix approach for each bug (after investigation)
- Whether to add error boundaries as a general fix
- Performance optimization strategy

</decisions>
