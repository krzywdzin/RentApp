Rental API endpoints:
- POST /rentals, GET /rentals(?status=X), GET /rentals/calendar(?from=&to=), GET /rentals/:id
- PATCH /rentals/:id/activate, PATCH /rentals/:id/return, PATCH /rentals/:id/extend, PATCH /rentals/:id/rollback

Contract API endpoints:
- POST /contracts, GET /contracts/:id, GET /contracts/rental/:rentalId, GET /contracts/:id/pdf, POST /contracts/:id/sign

State machine transitions (from Phase 3):
- DRAFT -> ACTIVE (activate)
- ACTIVE -> EXTENDED (extend), ACTIVE -> RETURNED (return)
- EXTENDED -> RETURNED (return)
- Admin rollback: ACTIVE -> DRAFT, EXTENDED -> ACTIVE, RETURNED -> EXTENDED/ACTIVE
