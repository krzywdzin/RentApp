## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline badge padding |
| sm | 8px | Compact element spacing, table cell padding vertical |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding within pages, sidebar section dividers |
| xl | 32px | Layout gaps between major content blocks |
| 2xl | 48px | Page-level vertical padding (content from top bar) |
| 3xl | 64px | Sidebar expanded width reserved space (sidebar width: 256px collapsed icon-only: 64px) |

Exceptions:
- Sidebar expanded width: 256px (not on scale, layout-specific)
- Sidebar collapsed width: 64px (icon-only mode)
- Table row height: 48px minimum for comfortable click targets
- Dashboard stat cards: 24px internal padding

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

**Notes:**
- Body at 14px suits data-dense admin panels (consistent with Linear/Vercel)
- Label at 12px / 400 for table headers, form labels, badges, sidebar items — differentiated from body by size alone, not weight
- Display at 28px for dashboard stat numbers and page titles
- All text renders in Inter; Polish characters (accented: a, c, e, l, n, o, s, z, z) fully supported

## Color

Uses shadcn zinc dark theme as base. All colors reference CSS custom properties from shadcn's theming system.

| Role | CSS Variable | Approximate HSL | Usage |
|------|-------------|-----------------|-------|
| Dominant (60%) | `--background` | 240 10% 3.9% (zinc-950) | Page background, main content area |
| Secondary (30%) | `--card` | 240 10% 7.8% (zinc-900) | Cards, sidebar background, table rows, dropdown menus |
| Accent (10%) | `--primary` | 0 0% 98% (zinc-50, white) | See reserved-for list below |
| Destructive | `--destructive` | 0 84% 60% (red-500) | Delete buttons, overdue badges, error states only |

**Additional semantic colors:**

| Token | Approximate HSL | Usage |
|-------|-----------------|-------|
| `--muted` | 240 3.7% 15.9% (zinc-800) | Muted text, disabled states, borders |
| `--muted-foreground` | 240 5% 64.9% (zinc-400) | Secondary text, placeholders |
| `--border` | 240 3.7% 15.9% (zinc-800) | Table borders, card borders, dividers |
| `--ring` | 240 4.9% 83.9% (zinc-300) | Focus ring on interactive elements |
| Success | 142 76% 36% (green-600) | Active/available status badges |
| Warning | 38 92% 50% (amber-500) | Expiring soon badges, extension needed alerts |

**Accent reserved for (exhaustive list):**
1. Primary action buttons ("Dodaj pojazd", "Zapisz", "Utworz wynajem")
2. Active sidebar navigation item indicator (left border or background highlight)
3. Pagination current page indicator
4. Toggle/switch active state
5. Link text on hover

Accent is NOT used for: table row hover (use `--muted` at 50% opacity), card borders, icons generally, status badges.

## Visual Focal Point

**Primary dashboard screen:** The stat card row is the primary visual anchor. The four cards sit at the top of the content area immediately below the top bar. When the overdue count is greater than 0, the "Przeterminowane" card renders its count in `--destructive` color and gains a subtle `--destructive` left border (3px), drawing first attention to urgent items. When all counts are zero, the row remains visually balanced with equal weight across cards.

**Entity list screens:** The data table is the focal point. The filter bar above it is secondary. The primary CTA button ("Dodaj pojazd", etc.) sits top-right of the page header and uses accent color to draw the eye after the table.

## Interaction Patterns

### Loading States
- **Table loading:** Skeleton rows (5 rows of skeleton cells matching column widths)
- **Card loading:** Skeleton with pulsing animation matching card dimensions
- **Detail page loading:** Skeleton for each section/tab content area
- **Action in progress:** Button shows spinner icon, disabled state, label changes to gerund ("Zapisywanie...", "Usuwanie...")

### Toast Notifications (sonner)
- **Success:** "Pojazd dodany pomyslnie", "Zmiany zapisane", "Wynajem utworzony"
- **Error:** Mirror the error state messages from above
- **Duration:** 4 seconds for success, 6 seconds for errors
- **Position:** Bottom-right

### Table Interactions
- **Row click:** Navigate to entity detail page
- **Row hover:** Subtle background highlight (`--muted` at 50% opacity)
- **Sort:** Click column header to toggle asc/desc, visual arrow indicator
- **Pagination:** Bottom of table, showing "Strona {n} z {total}" with prev/next buttons
- **Per-page:** Selector with options: 10, 25, 50

### Sidebar Behavior
- **Expanded (default):** 256px width, icon + label for each item
- **Collapsed:** 64px width, icon only with tooltip on hover
- **Toggle:** Chevron button at bottom of sidebar
- **Active indicator:** Left 3px border accent color on active item, background `--muted`

### Calendar Timeline (Wynajmy tab)
- **Y-axis:** Vehicle list (registration + make/model), one row per vehicle
- **X-axis:** Date timeline, scrollable horizontally
- **Rental blocks:** Colored bars spanning start-to-end date, color-coded by rental status
- **Conflict indication:** Red border/overlay on overlapping date ranges
- **Zoom levels:** Day view (default), Week view (toggle)
- **Click rental block:** Navigate to rental detail page
