**BFF proxy fix** (`apps/web/src/app/api/[...path]/route.ts`):

1. Read the incoming `Content-Type` header from `request.headers.get('content-type')`
2. If Content-Type starts with `multipart/form-data`:
   - Do NOT set `Content-Type: application/json` — instead pass through the original Content-Type header (which includes the boundary parameter)
   - Read body with `await request.arrayBuffer()` (not `.text()` — text encoding can corrupt binary data)
   - Forward the raw ArrayBuffer as the fetch body
3. For all other requests, keep existing JSON behavior unchanged
4. The proxy already handles non-JSON responses (lines 25-36 with arrayBuffer pass-through), so the response side is fine

Concrete change: replace the hardcoded headers + body logic with:
```typescript
const incomingContentType = request.headers.get('content-type') ?? '';
const isMultipart = incomingContentType.startsWith('multipart/form-data');

const headers: HeadersInit = {};
if (isMultipart) {
  headers['Content-Type'] = incomingContentType; // preserve boundary
} else {
  headers['Content-Type'] = 'application/json';
}
if (token) {
  headers['Authorization'] = `Bearer ${token}`;
}

let body: BodyInit | undefined;
if (['GET', 'HEAD'].includes(request.method)) {
  body = undefined;
} else if (isMultipart) {
  body = await request.arrayBuffer();
} else {
  body = await request.text();
}
```

**apiClient FormData support** (`apps/web/src/lib/api-client.ts`):

The apiClient currently hardcodes `Content-Type: application/json` in headers. When body is FormData, the browser must set Content-Type automatically (with boundary). Modify both the initial request and the retry-after-refresh to:
- If `options.body instanceof FormData`, do NOT set Content-Type header (let browser set it)
- Otherwise keep existing `Content-Type: application/json`

```typescript
const isFormData = options.body instanceof FormData;
const headers: HeadersInit = {
  ...(isFormData ? {} : { 'Content-Type': 'application/json' }),
  ...options.headers,
};
```

Apply same logic to the retry fetch (line 58-63).

**Import mutation hook** (`apps/web/src/hooks/queries/use-vehicles.ts`):

Add a new `useImportVehicles` hook:
```typescript
interface ImportResult {
  imported: number;
  skipped: number;
  errors: Array<{ row: number; field: string; message: string }>;
}

export function useImportVehicles() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: (file: File) => {
      const formData = new FormData();
      formData.append('file', file);
      return apiClient<ImportResult>('/vehicles/import', {
        method: 'POST',
        body: formData,
      });
    },
    onSuccess: (data) => {
      queryClient.invalidateQueries({ queryKey: vehicleKeys.all });
      toast.success(`Import zakonczony: ${data.imported} dodanych`);
    },
    onError: () => {
      toast.error('Wystapil blad podczas importu pojazdow');
    },
  });
}
```

Also export `ImportResult` type so the dialog component can use it.
