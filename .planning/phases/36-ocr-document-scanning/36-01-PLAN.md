    3. Create `apps/api/src/documents/documents.service.ts`:
    - Inject PrismaService and StorageService
    - `uploadPhoto(customerId, type, side, file, scannedById)`:
      - Build R2 key: `documents/${customerId}/${type}/${side}.jpg`
      - Upload file buffer via `this.storage.upload(key, file.buffer, file.mimetype)`
      - Generate thumbnail key: `documents/${customerId}/${type}/${side}_thumb.jpg` (use sharp to resize to 320px width)
      - Upsert CustomerDocument: find by `{ customerId_type: { customerId, type } }`, set `frontPhotoKey`/`frontThumbKey` or `backPhotoKey`/`backThumbKey` based on side, set `scannedById` and `scannedAt`
      - If previous key existed for this side, delete old R2 objects before uploading new
      - Return the upserted record
    - `getDocuments(customerId)`:
      - Find all CustomerDocuments for customer
      - For each, generate presigned download URLs for all non-null photo keys
      - Return array of CustomerDocumentDto with photo URLs
    - `deleteDocumentsByCustomerId(customerId)`:
      - Find all CustomerDocuments, collect all R2 keys
      - Delete R2 objects, then delete DB records
      - Used for cascade cleanup with retention

    4. Create `apps/api/src/documents/documents.controller.ts`:
    - `POST /customers/:customerId/documents/:type/:side` with `@UseInterceptors(FileInterceptor('file'))`

      - Calls documentsService.uploadPhoto
    - `GET /customers/:customerId/documents`
      - Roles: ADMIN (workers see documents only during scanning, not via API)
      - Calls documentsService.getDocuments
