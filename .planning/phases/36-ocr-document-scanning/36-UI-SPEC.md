## Spacing Scale

Inherited from `apps/mobile/src/lib/theme.ts` (locked legacy tokens). No new tokens introduced.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, label-to-input margin |
| sm | 8px | Inline icon padding, compact element spacing |
| md | 12px | Inner form group gaps (between label groups) |
| base | 16px | Default element spacing, input horizontal padding |
| lg | 20px | Between form sections |
| xl | 24px | Button horizontal padding, major section gaps |
| xxl | 32px | Page-level horizontal padding |
| section | 40px | Top/bottom page section breaks |

Exceptions: Inherited legacy exceptions — not modifiable in Phase 36 without breaking existing components across 33+ completed phases: md (12px), lg (20px), section (40px). These tokens are defined in `apps/mobile/src/lib/theme.ts` and consumed by AppInput, AppButton, AppCard, WizardStepper, and numerous other shared components; changing them would require a cross-phase design migration. Mobile touch targets at 48px height (AppButton minHeight, input height). Camera shutter button at 72px diameter (industry standard for capture buttons). Document guide overlay frame uses 85.6mm:54mm aspect ratio (ISO/IEC 7810 ID-1 standard, approximately 1.586:1).

**Accent (forest-green) reserved for:**
- "Skanuj dowod osobisty" and "Skanuj prawo jazdy" scan buttons
- "Zatwierdz" confirmation button on OCR review screen
- "Zaktualizuj" button on diff view for existing customers
- Camera shutter button ring
- Checkbox active state in diff view

**Warning (amber-glow) reserved for:**
- OCR field border when value was extracted but has low confidence (partially recognized)
- Guide overlay frame border during camera positioning

**Terracotta-tint reserved for:**
- OCR field background when field could not be extracted (empty, requires manual entry)

**Sage-tint reserved for:**
- OCR field background when value was successfully extracted

## Visual Hierarchy

### Scan Buttons in Customer Step (mobile)

1. **Position**: Below existing customer personal data fields (after idExpiryDate), before address section
2. **Layout**: Two full-width buttons stacked vertically, 12px gap between them
3. **Button style (no scan yet)**: Secondary variant -- outlined border (forest-green), camera icon left-aligned, 48px height
4. **Button style (scan complete)**: Compact card -- 64px thumbnail left, "Zeskanowano" label + "Skanuj ponownie" text link right, warm-stone background, rounded corners

### Guide Overlay (mobile)

1. **Background**: Full-screen black overlay at 0.85 opacity
2. **Document frame**: Transparent cutout, 1.586:1 aspect ratio, centered vertically offset 80px up from center, 2px border in amber-glow, rounded corners 8px
3. **Instruction text**: Below frame, 24px gap, white text, 16px/600, centered
4. **Step indicator**: Below instruction, 8px gap, "Przod (1/2)" or "Tyl (2/2)", white text, 13px/400
5. **Capture button**: Bottom center, 40px from bottom safe area. 72px circle, forest-green fill, white camera icon 32px
6. **Torch button**: Top-right corner, 16px from edges, 44px touch target, white icon, semi-transparent background

### Confirmation Screen (mobile, new customer)

1. **Screen**: Full-screen modal with cream background, scroll view
2. **Thumbnail**: Top of screen, centered, 120px height, auto width maintaining aspect ratio, 8px border-radius, 1px warm-stone border
3. **Section heading**: "Dane z dowodu osobistego" -- 20px/600 Fraunces-Variable, left-aligned, 24px below thumbnail
4. **Field list**: Standard AppInput fields, 12px vertical gaps between field groups
5. **OCR success fields**: sage-tint (#F0F7F4) background on the input container
6. **OCR failed fields**: terracotta-tint (#FDF5F3) background on the input container, helper text "Nie odczytano -- wpisz recznie" in 13px/400 terracotta color
7. **OCR low-confidence fields**: amber-glow (#D4A853) 1px border on input, helper text "Sprawdz poprawnosc" in 13px/400 warm-gray
8. **Address section**: Separated by section heading "Adres -- wpisz recznie", with helper text below heading in 13px/400 warm-gray: "Nowy dowod nie zawiera adresu. Wpisz dane recznie."
9. **Bottom action bar**: Sticky at bottom, cream background, 16px padding, "Zatwierdz" primary full-width + "Odrzuc skan" text link below (terracotta, centered, 16px/400)

### Diff View (mobile, existing customer)

1. **Screen**: Full-screen modal with cream background, scroll view
2. **Screen heading**: "Porownanie danych" -- 20px/600 Fraunces-Variable, 24px top padding
3. **Column headers**: Fixed row below heading -- "Obecne" left (warm-gray, 13px/600), "Ze skanu" right (forest-green, 13px/600)
4. **Field rows**: Each field is a row with checkbox (left, 24px), current value (center-left, 16px/400 warm-gray), OCR value (center-right, 16px/400 charcoal). Alternating row backgrounds: cream / warm-stone. Row height: 48px minimum
5. **Changed fields**: OCR value in charcoal (normal weight). Row has sage-tint background
6. **Unchanged fields**: Both values identical, row has cream background, checkbox disabled and unchecked
7. **Bottom action bar**: Sticky, "Zaktualizuj zaznaczone" primary full-width button, "Zachowaj obecne dane" text link below

### Documents Section (web, customer detail)

1. **Section heading**: "Dokumenty" -- 20px/600 Fraunces, left-aligned, consistent with other customer detail sections
2. **Grid**: 2 columns (Dowod osobisty | Prawo jazdy), each column with 2 thumbnail rows (Przod | Tyl)
3. **Thumbnail**: 160px wide, auto height, rounded-lg, warm-stone border 1px, hover: ring-2 forest-green, cursor pointer
4. **Sub-labels**: "Przod" / "Tyl" in 14px/400 muted-foreground above each thumbnail
5. **Empty placeholder**: Dashed border box, 160px wide, 100px height, centered text "Brak" in muted-foreground
6. **Image dialog**: shadcn Dialog, max-width 640px, image scales to fit, no actions (view only)

## Interaction Contracts

### Scan Flow State Machine (mobile)

```
IDLE -> FRONT_GUIDE -> FRONT_CAPTURE -> BACK_GUIDE -> BACK_CAPTURE -> PROCESSING -> REVIEW
```

1. **IDLE**: Scan button visible in customer step. Tap opens FRONT_GUIDE.
2. **FRONT_GUIDE**: Guide overlay modal appears. User taps "Zrob zdjecie".
3. **FRONT_CAPTURE**: `ImagePicker.launchCameraAsync({ quality: 0.8, allowsEditing: false })`. If canceled, return to IDLE. If captured, store front URI, transition to BACK_GUIDE.
4. **BACK_GUIDE**: Guide overlay updates instruction text to "Umiesc tyl..." and step to "Tyl (2/2)". User taps "Zrob zdjecie".
5. **BACK_CAPTURE**: Same camera launch. If canceled, keep front photo, still transition to PROCESSING (back is optional but encouraged). If captured, store back URI.
6. **PROCESSING**: Show spinner with "Odczytywanie danych..." Run `extractTextFromImage()` on front URI (and back URI if captured). Parse results with `parseIdCard()` or `parseDriverLicense()`. If total OCR returns zero usable fields, show error toast and return to IDLE.
7. **REVIEW**: If new customer -> DocumentConfirmation screen. If existing customer -> DocumentDiffView screen.

### Confirmation Screen Interactions (new customer)

1. All OCR-extracted fields are pre-filled in editable AppInput fields
2. Fields with no extraction are empty with terracotta-tint background
3. Address fields are always empty (modern Polish ID has no address) with helper text
4. Worker edits any field freely -- standard keyboard input
5. "Zatwierdz" button: validates required fields (firstName, lastName required; others optional), stores confirmed values in draft store, dismisses modal, returns to customer step with fields populated
6. "Odrzuc skan" link: discards all OCR data, returns to customer step unchanged

### Diff View Interactions (existing customer)

1. Each field row shows current value (left) vs OCR value (right) with a checkbox
2. Fields where OCR value differs from current: checkbox enabled and pre-checked
3. Fields where values are identical: checkbox disabled, row visually muted
4. Fields where OCR returned null: row not shown (no comparison possible)
5. Worker can uncheck fields they do not want to update
6. "Zaktualizuj zaznaczone": applies only checked field values to draft store, dismisses modal
7. "Zachowaj obecne dane": discards OCR results, returns to customer step unchanged

### Re-scan Flow (mobile)

1. If scan already exists for this document type, button shows thumbnail + "Skanuj ponownie"
2. Tapping "Skanuj ponownie" shows ConfirmationDialog: "Poprzednie zdjecia i dane ze skanu zostana zastapione."
3. "Skanuj ponownie" (confirm): clears previous scan data, restarts scan flow from FRONT_GUIDE
4. "Nie, zachowaj" (cancel): dismisses dialog, keeps existing scan

### Camera Permissions (mobile)

1. Before launching camera, call `ImagePicker.requestCameraPermissionsAsync()`
2. If denied, show toast: "Brak uprawnien do kamery" / "Wlacz dostep do kamery w ustawieniach"
3. Do NOT open settings automatically (follow established pattern from photos.tsx)

### Torch Toggle (mobile, guide overlay)

1. Torch button in guide overlay top-right corner
2. Tap toggles flash mode between "off" and "on"
3. Flash mode is passed to `ImagePicker.launchCameraAsync()` via `cameraType` options (if expo-image-picker supports it; otherwise document as future enhancement)
4. Icon: flashlight-off / flashlight (lucide) or custom torch icon

### Document Photos Upload (mobile -> API)

1. After "Zatwierdz" on confirmation screen, document photos are queued for upload
2. Upload happens during rental submission (not immediately) -- follows vehicle photo pattern
3. Photos uploaded to `/customers/:customerId/documents/:type/:side` endpoint
4. Upload failure shows toast error but does not block rental creation (photos can be re-uploaded)

### Admin Document Viewing (web)

1. Customer detail page shows "Dokumenty" section below existing customer data
2. Thumbnails loaded via presigned R2 URLs
3. Click thumbnail opens Dialog with full-resolution image
4. No edit/delete actions in web panel (document management is mobile-only)
5. Skeleton loading state while thumbnails load
