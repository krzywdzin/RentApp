  <action>
**1. Create RentalDrivers controller** (`rental-drivers.controller.ts`):
- `POST /rentals/:rentalId/driver` -- creates second driver (body: CreateRentalDriverDto). Requires auth. Returns created driver DTO.
- `GET /rentals/:rentalId/driver` -- gets second driver for rental. Returns driver DTO or 404.
- `DELETE /rentals/:rentalId/driver` -- deletes second driver. Returns 204.
- Use `@UseGuards(JwtAuthGuard)` on all endpoints (same pattern as other controllers).
- Validate rentalId exists before creating driver.

**2. Update RentalDriversModule:** Add controller to controllers array. Import CepikModule if needed for the verify endpoint, OR add verify endpoint to CepikController.

**3. Extend CEPiK service** (`cepik.service.ts`):
- Add new method: `verifyDriver(driverId: string, rentalId: string, checkedById: string): Promise<CepikVerification>`
- This method:
  a. Fetches RentalDriver by id, decrypts licenseNumber
  b. Calls the same CEPiK API verification logic as existing `verify()` method
  c. Creates CepikVerification record with `driverId` set (instead of customerId)
  d. `customerId` can be null for driver verifications
- Extract shared verification logic into a private method to avoid duplication.

**4. Extend CEPiK controller** (`cepik.controller.ts`):
- Add endpoint: `POST /cepik/verify-driver` with body `{ driverId: string, rentalId: string }`
- Create `VerifyDriverDto` with `@IsUUID()` for both fields
- Calls `cepikService.verifyDriver()`
- Same auth guard pattern as existing verify endpoint

**5. Update CepikModule** if needed: ensure RentalDriversModule is imported so CepikService can access RentalDriversService for decrypting driver license number.

IMPORTANT: The CEPiK API is called with the driver's license number. For second drivers, this comes from the encrypted RentalDriver record, not from Customer. The service needs RentalDriversService injected to decrypt the license number.
