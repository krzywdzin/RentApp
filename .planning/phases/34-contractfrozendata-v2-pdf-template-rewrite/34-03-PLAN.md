**6. Add PATCH /rentals/:id endpoint for rentalTerms and termsNotes:**

Create `update-rental-terms.dto.ts`:
```typescript
import { IsString, IsOptional, MaxLength } from 'class-validator';

export class UpdateRentalTermsDto {
  @IsOptional()
  @IsString()
  @MaxLength(100000)
  rentalTerms?: string;

  @IsOptional()
  @IsString()
  @MaxLength(10000)
  termsNotes?: string;
}
```

Add to `rentals.controller.ts`:
```typescript
@Patch(':id/terms')
@UseGuards(JwtAuthGuard)
async updateTerms(
  @Param('id', ParseUUIDPipe) id: string,
  @Body() dto: UpdateRentalTermsDto,
) {
  return this.rentalsService.updateTerms(id, dto);
}
```

Add to `rentals.service.ts`:
```typescript
async updateTerms(id: string, dto: UpdateRentalTermsDto) {
  return this.prisma.rental.update({
    where: { id },
    data: {
      ...(dto.rentalTerms !== undefined && { rentalTerms: dto.rentalTerms }),
      ...(dto.termsNotes !== undefined && { termsNotes: dto.termsNotes }),
    },
  });
}
```

This endpoint is required by Plan 05 (mobile) -- before contract creation, if the worker customized terms, the mobile app PATCHes the rental to set rentalTerms/termsNotes. The contract service then reads these fields when building frozen data.

IMPORTANT: The CEPiK API is called with the driver's license number. For second drivers, this comes from the encrypted RentalDriver record, not from Customer. The service needs RentalDriversService injected to decrypt the license number.
