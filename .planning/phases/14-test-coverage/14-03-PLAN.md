1. First, run the existing API tests with coverage to determine the current baseline:
   ```
   cd apps/api && npx jest --coverage --coverageReporters=text-summary 2>&1 | tail -10
   ```
   Note the current statement coverage percentage. Use this to set a threshold slightly below current (e.g., if current is 72%, set threshold to 60% as a reasonable floor that won't break but prevents major regression).

2. Update apps/api/jest.config.ts to add coverageThreshold:
   ```ts
   import type { Config } from 'jest';
   const config: Config = {
     moduleFileExtensions: ['js', 'json', 'ts'],
     rootDir: 'src',
     testRegex: '.*\\.spec\\.ts$',
     transform: { '^.+\\.(t|j)s$': 'ts-jest' },
     collectCoverageFrom: ['**/*.(t|j)s'],
     coverageDirectory: '../coverage',
     testEnvironment: 'node',
     coverageThreshold: {
       global: {
         statements: 60,
       },
     },
   };
   export default config;
   ```
   NOTE: Adjust the 60 to be at or slightly below the measured baseline. The CONTEXT.md suggests 60% as a starting point. If actual coverage is lower than 60%, use the actual value minus 5% rounded down. If actual coverage is higher (e.g., 75%), still use 60% -- it's a floor to prevent regression, not a target.

3. Update apps/api/package.json test script to include coverage:
   - Change "test": "jest" to "test": "jest --coverage"
   - Keep "test:e2e" unchanged
   - Add "test:no-coverage": "jest" for quick runs without coverage overhead

4. Verify: Run `cd apps/api && npm test` and confirm it passes with coverage report showing threshold enforcement.
