In `apps/mobile/src/components/SignatureScreen.tsx`, the current `useEffect` (lines 41-51) has `stepLabel` in its dependency array, causing the orientation lock/unlock cycle to re-run on every step change. This causes flickering and does NOT explicitly clear the canvas.

Split this into TWO separate effects:

**Effect 1 -- Orientation lock (mount/unmount only, empty dependency array):**
```typescript
useEffect(() => {
  ScreenOrientation.lockAsync(
    ScreenOrientation.OrientationLock.LANDSCAPE_LEFT,
  );

  return () => {
    ScreenOrientation.lockAsync(
      ScreenOrientation.OrientationLock.PORTRAIT_UP,
    );
  };
}, []);
```

**Effect 2 -- Clear canvas on step change (depends on stepLabel):**
```typescript
useEffect(() => {
  // Clear the canvas when moving to the next signature step
  signatureRef.current?.clearSignature();
}, [stepLabel]);
```

Keep all other code unchanged -- `handleClear`, `handleConfirm`, `handleOK`, `handleEmpty`, JSX, and styles remain exactly as-is.
