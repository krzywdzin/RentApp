# Plan 15-02: Rate Limiting, CORS Configuration & Security Headers

**Phase:** 15 — API Hardening & Security
**Requirements:** APIS-01, APIS-02, APIS-03, APIS-04, APIS-05

## Tasks

### Wave 1: Security Controls (parallel)

**Task 1.1: Rate limiting with per-endpoint config**
- Register ThrottlerGuard as APP_GUARD in app.module.ts
- Default: 100 requests per 60s (already configured)
- Override for auth endpoints: @Throttle({ default: { limit: 10, ttl: 60000 } }) on AuthController
- Ensure @Public endpoints still get rate limited (ThrottlerGuard runs independently of JwtAuthGuard)

**Task 1.2: CORS from environment**
- Read `CORS_ORIGINS` env var (comma-separated list)
- Default to `http://localhost:3001` in development
- Remove hardcoded `http://192.168.0.26:3001` from main.ts
- Add CORS_ORIGINS to env.validation.ts as optional with default

**Task 1.3: Security headers & body size**
- Configure Helmet for production: enable HSTS, CSP (if applicable), X-Frame-Options DENY
- Keep existing 10MB limit for signature uploads
- Add 1MB default limit consideration (already handled by express defaults, just document)

**Task 1.4: Ensure no secret defaults in production**
- In env validation, when NODE_ENV=production, require: JWT_ACCESS_SECRET, JWT_REFRESH_SECRET, FIELD_ENCRYPTION_KEY, DATABASE_URL, REDIS_URL
- Log warning if SMSAPI_TOKEN is missing (non-critical but notifications won't work)

## Success Criteria
- [ ] Auth endpoints return 429 after 10 rapid requests
- [ ] CORS_ORIGINS env var controls allowed origins
- [ ] Helmet headers present in responses
- [ ] Production mode requires all secrets
