# Plan 15-01: Environment Validation, Graceful Shutdown & Global Error Filter

**Phase:** 15 — API Hardening & Security
**Requirements:** APIH-01, APIH-02, APIH-03, APIH-04

## Tasks

### Wave 1: Core Infrastructure (parallel)

**Task 1.1: Environment validation at startup**
- Create `apps/api/src/common/env.validation.ts`
- Define required env vars: DATABASE_URL, JWT_ACCESS_SECRET, JWT_REFRESH_SECRET, REDIS_URL
- Define optional with defaults: PORT, NODE_ENV, S3_ENDPOINT, CORS_ORIGINS
- Validate at bootstrap before app creation — fail fast with clear error listing missing vars
- Wire into `main.ts` bootstrap

**Task 1.2: Global exception filter**
- Create `apps/api/src/common/filters/all-exceptions.filter.ts`
- Implement `ExceptionFilter` that catches all exceptions
- Return consistent JSON: `{ statusCode, message, error, timestamp, path }`
- Log errors with NestJS Logger
- Handle HttpException (pass through), Prisma errors (map to 4xx/5xx), unknown errors (500)
- Register in `main.ts` via `app.useGlobalFilters()`

**Task 1.3: Graceful shutdown**
- Add `app.enableShutdownHooks()` in `main.ts`
- Implement `OnModuleDestroy` in PrismaService (disconnect DB)
- Log shutdown sequence
- Handle SIGTERM and SIGINT

### Wave 2: PDF Error Handling & DRAFT→ACTIVE Fix

**Task 2.1: PDF generation error handling**
- Wrap PDF generation in contracts.service.ts with try/catch
- Return 500 with descriptive message on PDF failure instead of crashing
- Add retry logic (1 retry) for transient Puppeteer failures

**Task 2.2: Verify DRAFT→ACTIVE transition**
- Review contracts.service.ts sign() method — confirm DRAFT→ACTIVE triggers after 4th signature
- Ensure idempotent — re-signing already-signed type doesn't duplicate transition
- Add guard: only transition if current rental status is DRAFT

## Success Criteria
- [ ] API fails to start when DATABASE_URL is missing
- [ ] Unhandled exception returns JSON, not HTML
- [ ] SIGTERM results in clean shutdown
- [ ] PDF generation failure returns 500 JSON error
