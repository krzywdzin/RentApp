| Stat | Format | Example |

## Interaction Contracts

### Settlement List Tab

1. Tab switch to "Rozliczenia" loads settlement data (same useRentals hook, filtered client-side by settlementStatus or via server query param).
2. Filter changes reset pagination to page 0 (same pattern as existing RentalFilterBar).
3. Row click navigates to `/wynajmy/{id}` rental detail page (same as Lista tab).
4. Summary bar updates reactively when filters change (computed from filtered dataset).

### Settlement Form in Rental Detail

1. Form loads pre-populated from rental.settlementStatus, rental.settlementAmount, rental.settlementNotes.
2. Amount input accepts decimal PLN values (e.g., "150,50"). Convert to grosze (15050) before PATCH.
3. "Zapisz rozliczenie" calls PATCH `/rentals/:id/settlement`. On success: toast + invalidate rental query. On error: error toast.
4. settledAt auto-displays when status is ROZLICZONY (set server-side, not editable).
5. No confirmation dialog needed -- settlement changes are reversible per locked decision.

### VAT Modal in Mobile Return Flow

1. Modal renders on mount of return screen when `rental.vatPayerStatus` is `FULL_100` or `HALF_50`.
2. Modal blocks interaction with underlying screen (transparent overlay with pointer events).
3. Single "Rozumiem" button dismisses modal via `useState(false)` -- no persistence.
4. Modal does NOT re-appear on back-navigation within the same return session (state lives in component, not route).
5. If worker leaves return flow entirely and re-enters, modal shows again (state resets with component unmount).
