## Spacing Scale

Declared values from existing `apps/mobile/src/lib/theme.ts`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, stepper top padding |
| md | 12px | Card margins, internal gaps |
| base | 16px | Default horizontal padding, section gaps |
| lg | 20px | PDF margins (mapped to 15-20mm) |
| xl | 24px | Section padding |
| xxl | 32px | Layout gaps |
| section | 40px | Major section breaks |

**Spacing Exceptions:** Values 12px (md), 20px (lg), and 40px (section) are pre-existing project tokens defined in `apps/mobile/src/lib/theme.ts`, established before Phase 39. All three are multiples of 4 and are approved project conventions used across the application. For new Phase 39 components, restrict to standard-set values only: {4, 8, 16, 24, 32, 48, 64}. Use existing theme tokens only when composing with pre-existing components that already use them (e.g., AppCard internal padding at 12px).

PDF template uses mm-based spacing (20mm page padding, 4mm field spacing, 25mm signature block top margin) — these mirror the client template layout, not the mobile spacing scale.

## Component Inventory

### Mobile — New Screens

#### 1. Protocol Form Screen (`protocol.tsx`)

- **WizardStepper:** currentStep=5, totalSteps=8
- **Step title:** "Protokol zwrotu" (Fraunces 20px/600)
- **Auto-filled fields (read-only):** Displayed in an AppCard (warmStone background)
  - Zdajacy: customer full name
  - Data i godzina zwrotu: formatted current date/time
  - Marka i model: vehicle make + model
  - Nr rejestracyjny: vehicle plate
  - Miejsce odbioru: return location address or "Nie podano"
- **Cleanliness chip selector:** 3 horizontal chips in a row
  - Chips: "Czysty" | "Brudny" | "Do mycia"
  - Unselected: warmStone background, charcoal text, 1px sand border, borderRadius 8px
  - Selected: forestGreen background, cream text, no border
  - Chip height: 40px, horizontal padding: 16px, gap between chips: 8px
  - Touch target: minimum 44px height (40px chip + 4px vertical margin)
- **Cleanliness note:** Single-line TextInput, visible only when cleanliness is selected
  - Placeholder: "Uwagi do czystosci (opcjonalne)"
  - Style: bottom border only (sand), same as notes.tsx textarea pattern
- **Inne (free-text):** Multiline TextInput
  - Placeholder: "Dodatkowe uwagi (opcjonalne)"
  - minHeight: 120px, bottom border only (sand)
- **Bottom bar:** AppButton "Dalej" fullWidth, pinned to bottom with sand top border

#### 2. Customer Signature Screen (`protocol-sign-customer.tsx`)

- **WizardStepper:** currentStep=6, totalSteps=8
- **Reuses SignatureScreen component** with:
  - title: "Podpis zdajacego"
  - stepLabel: "Krok 6 z 8"
  - instruction: "Klient podpisuje protokol zwrotu"
- Landscape orientation, full-width canvas
- "Potwierdz" and "Wyczysc" buttons (existing SignatureScreen UI)

#### 3. Worker Signature Screen (`protocol-sign-worker.tsx`)

- **WizardStepper:** currentStep=7, totalSteps=8
- **Reuses SignatureScreen component** with:
  - title: "Podpis odbierajacego"
  - stepLabel: "Krok 7 z 8"
  - instruction: "Pracownik podpisuje protokol zwrotu"
- Same layout as customer signature screen

#### 4. Confirm Screen Updates (`confirm.tsx`)

- **WizardStepper:** totalSteps changes from 5 to 8, currentStep=8
- **New AppCard section** added after existing notes card:
  - Section label: "Protokol zwrotu"
  - Shows: cleanliness value (chip label text), cleanliness note (if any), other notes (if any)
  - Shows: checkmark icons or "Podpisano" text for each collected signature

### Web Admin — Rental Detail Update

#### Protocol Download Section

- **Location:** New section in rental detail page, visible only when rental status is RETURNED and protocol exists
- **Component:** shadcn Card with CardHeader + CardContent
  - CardTitle: "Protokol zwrotu"
  - Content: shadcn Button with download icon (lucide `Download`)
  - Button label: "Pobierz protokol"
  - Button variant: `outline`
  - Shows "Wygenerowano: {date}" below the button in muted text (text-muted-foreground, text-sm)

### PDF Template

- **Single page A4** (210mm x 297mm)
- **Margins:** 20mm all sides (matching client template visual spacing)
- **Header block** (top-left, no logo):
  - Line 1: "KITEK Pawel Romanowski" (13px, weight 700)
  - Line 2: "www.p-romanowski.pl" (11px, color #0066CC, underlined)
  - Line 3: "Tel. (+48) 602 367 100" (11px)
  - Bottom margin: 10mm
- **Title:** "PROTOKOL ZDAWCZO - ODBIORCZY SAMOCHODU" (18px, weight 700)
  - Top margin: 15mm from header, bottom margin: 10mm
- **7 numbered fields:** Each on its own line, 4mm vertical spacing
  - Number + label in bold (13px/700): "1. Zdajacy:"
  - Value inline after label (13px/400)
  - No dotted lines for filled values (dotted lines are the paper template's blank spaces)
- **Signature block:** Flex row, space-between, 25mm top margin
  - Two columns at 45% width each
  - Signature image: max-height 20mm, max-width 100%, centered
  - Dotted line below signature image (1px dotted #333, top border)
  - Label below line: "Zdajacy (data i podpis)" / "Odbierajacy (data i podpis)" (11px)

## Interaction States

### Cleanliness Chip Selector

| State | Visual |
|-------|--------|
| None selected (initial) | All 3 chips in warmStone, charcoal text. "Dalej" button disabled. |
| One selected | Selected chip: forestGreen bg, cream text. Others: warmStone, charcoal. "Dalej" enabled. |
| Tap selected chip | Deselects (returns to none selected). "Dalej" disabled. |

### Protocol Form Validation

| Condition | Behavior |
|-----------|----------|
| Cleanliness not selected | "Dalej" button disabled (grayed out). No error toast -- implicit guidance via disabled state. |
| Cleanliness selected, notes empty | Valid. Notes are optional. |
| All fields filled | "Dalej" enabled, navigates to customer signature. |

### Signature Screens

| State | Visual |
|-------|--------|
| Canvas empty | "Potwierdz" button disabled |
| Canvas has strokes | "Potwierdz" button enabled (forestGreen) |
| "Wyczysc" tapped | Canvas cleared, "Potwierdz" disabled again |
| "Potwierdz" tapped | Haptic feedback, base64 captured, navigate to next step |

### Confirm Screen — Submit Flow

| State | Visual |
|-------|--------|
| Submitting | AppButton shows loading spinner, disabled |
| Success | Toast "Zwrot zarejestrowany", navigate to rentals list |
| Protocol creation error | Toast "Nie udalo sie utworzyc protokolu zwrotu. Sprobuj ponownie." Button re-enabled. |
| Return API error | Toast "Blad podczas zwrotu pojazdu" (existing error). Button re-enabled. |

### Web Admin — Protocol Section

| State | Visual |
|-------|--------|
| Rental not RETURNED | Section not rendered |
| RETURNED, no protocol | Section not rendered |
| RETURNED, protocol exists | Card with download button visible |
| Download clicked | Opens PDF in new tab (presigned URL) |
