# Phase 6 — UI Design Contract

> Visual and interaction contract for the Mobile App (Expo / React Native). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Mobile-only phase.** This phase targets field employees on Android and iOS devices. All styling uses NativeWind (Tailwind CSS for React Native). No shadcn -- this is React Native, not web.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (React Native -- no shadcn) |
| Component library | React Native core + Expo modules |
| Styling | NativeWind v4 (Tailwind CSS for React Native) |
| Icon library | lucide-react-native |
| Font | System default (SF Pro on iOS, Roboto on Android) |
| Navigation | expo-router (file-based routing, bottom tabs) |
| Forms | react-hook-form + @hookform/resolvers/zod + @rentapp/shared schemas |

**Light mode only** (CONTEXT.md decision). Dark mode deferred.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline badge padding |
| sm | 8px | Compact element spacing, list item internal gaps |
| md | 16px | Default element spacing, card padding, form field gaps, screen horizontal padding |
| lg | 24px | Section padding, wizard step content padding |
| xl | 32px | Layout gaps between major content blocks |
| 2xl | 48px | Dashboard section breaks, wizard header height |
| 3xl | 64px | Bottom tab bar height (including safe area inset) |

Exceptions:
- Touch targets: 48px minimum height/width for all tappable elements (exceeds Apple HIG 44pt, stays on 4px grid)
- Signature canvas: full-screen, 0px padding (landscape orientation)
- Screen horizontal padding: 16px (md) consistently on all screens
- Bottom tab bar: 64px height (3xl) including safe area padding
- Wizard progress indicator: 4px height bar with 8px vertical margin
- Bottom sheet handle: 4px height, 32px width, centered with 16px top padding

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 13px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.1 |

**Notes:**
- 2 weights only: 400 (regular) for body and labels, 600 (semibold) for headings and display numbers
- Body at 16px for readability in outdoor/field conditions (not 14px like admin panel)
- Label at 13px for form labels, tab bar labels, status badges, list item secondary text
- Heading at 20px for screen titles, wizard step titles, section headers
- Display at 28px for dashboard stat numbers only (active rentals count, today's pickups count)
- System fonts only -- SF Pro on iOS, Roboto on Android. No custom font loading needed.
- Polish characters fully supported by both system fonts

1. Primary CTA buttons ("Rozpocznij wynajem", "Dalej", "Podpisz", "Zatwierdz zwrot")

3. Wizard progress bar filled segment
4. Active step indicator dot
5. Links and interactive text elements
6. FAB / quick action button on dashboard
7. Focused input border ring

Accent is NOT used for: status badges (use success/warning/overdue), card backgrounds, navigation headers, list item backgrounds, dividers.

## Visual Focal Point

### Dashboard
The stat cards row is the primary visual anchor. Three cards with large display numbers (28px semibold) sit in a horizontal scroll at the top below the greeting. Below them, the two quick-action buttons ("Nowy wynajem" in accent fill, "Szybki zwrot" in outline) draw the eye as the primary calls to action. The upcoming returns list fills the remaining scroll area.

### Rental Wizard
The WizardStepper progress bar is the persistent visual anchor at the top of every step. It uses a filled bar (accent color) showing progress fraction, with step dots below. The step content area below is a scrollable form. The primary CTA button is fixed at the bottom of the screen (above safe area), always visible without scrolling.

### Signature Screen
The canvas occupies the center of the landscape screen. White background with a subtle bottom border line indicating the signing baseline. The "Zatwierdz podpis" button sits bottom-right, "Wyczysc podpis" bottom-left. The header bar with signature label and step count sits at the top in a compact 48px bar.

### Rentals List
The search bar sits at the top, followed by horizontal filter chips. The rental list items are the focal point -- each card shows the most important info at a glance: customer name (semibold weight), vehicle registration (regular weight, zinc-500), date range, and a color-coded status badge on the right edge.

## Interaction Patterns

### Authentication Flow

1. **Launch:** Splash screen with KITEK logo (configurable brand asset).
2. **First login:** Email + password form. On success, store tokens in expo-secure-store. Prompt biometric enrollment.
3. **Subsequent launches:** If biometric enabled, prompt immediately. Fallback to password form on failure/decline.
4. **Token refresh:** Silent background refresh via TanStack Query interceptor. 401 response triggers refresh attempt. Failed refresh redirects to login screen.
5. **Logout:** Clears secure store, resets navigation to login.

### Rental Creation Wizard

1. **Entry:** Tap "Nowy wynajem" on dashboard or tab. Opens step 1.
2. **Step navigation:** Forward with "Dalej" button (bottom-fixed). Back with header back arrow. Steps validate before advancing (Zod schema per step).
3. **Draft persistence:** Zustand store with async persist to device storage. On re-entry, prompt: "Masz niezakonczony wynajem. Kontynuowac?" with "Tak" / "Zacznij od nowa".
4. **Customer inline creation:** "Nowy klient" opens bottom sheet with customer form. On save, customer is selected and sheet closes. Wizard state preserved.
5. **Vehicle selection:** Cards show registration, make/model, year. Only AVAILABLE vehicles shown by default. Toggle "Tylko dostepne" to see all.
6. **Date/time pickers:** Native date/time pickers (DateTimePicker from @react-native-community/datetimepicker). Start date defaults to now. End date defaults to start + 1 day.
7. **Contract preview:** Scrollable read-only view showing all contract data assembled from steps 1-3. RODO checkbox must be checked to enable "Przejdz do podpisow".
8. **Signature capture:** Device rotates to landscape. 4 sequential signature screens. Each signature uploaded immediately after confirm. Retry with exponential backoff on failure (3 attempts, then show error).
9. **Success:** Device returns to portrait. Success screen with checkmark animation. Draft cleared from storage.

### Vehicle Return Wizard

1. **Entry:** From rental detail screen, tap "Rozpocznij zwrot" (only visible for ACTIVE/EXTENDED rentals).
2. **Step navigation:** Same pattern as rental wizard -- forward validation, bottom-fixed CTA, back arrow.
3. **Mileage step:** Numeric keyboard input. Handover mileage displayed for reference. Validation: return mileage must be >= handover mileage.
4. **Checklist:** Toggle switches for standard items (bodywork, interior, tires, lights, documents). Each item has optional notes field that expands on toggle-on.
5. **Submit:** Review screen shows summary. "Zatwierdz zwrot" triggers API call. On success, rental transitions to RETURNED status.

### Offline Behavior

- **Detection:** NetInfo listener. When offline, yellow OfflineBanner appears below status bar.
- **Read access:** TanStack Query cache serves previously loaded data. Lists and details remain browsable.
- **Write access:** All mutation buttons disabled with reduced opacity. Toast: "Brak polaczenia" on tap attempt.
- **Recovery:** Banner disappears when connectivity returns. Stale queries refetch automatically.

### Pull-to-Refresh

- All list screens (dashboard upcoming returns, rentals list) support pull-to-refresh.
- RefreshControl with accent color spinner.
- Triggers TanStack Query refetch of visible queries.

### Loading States

- **Screen load:** Skeleton placeholders matching content layout (cards as rounded rectangles, list items as horizontal bars).
- **Button loading:** Spinner replaces button text. Button disabled during loading. No double-tap.
- **Signature upload:** Per-signature circular progress indicator overlaid on canvas thumbnail in success sequence.
- **Wizard step load:** Skeleton in content area, stepper visible and static.

### Toast/Feedback

- **Rental created:** "Wynajem utworzony pomyslnie" toast, success (green), 3 seconds.
- **Return submitted:** "Zwrot zarejestrowany" toast, success (green), 3 seconds.
- **Signature captured:** Brief haptic feedback (Haptics.impactAsync light). No toast.
- **Draft resumed:** "Wznowiono szkic wynajmu" toast, info (zinc), 3 seconds.
- **Error:** Persistent error toast until dismissed, with "Ponow" (Retry) action when applicable.
