       - WizardStepper at top: currentStep=1, totalSteps=5
       - Screen title: "Potwierdzenie wynajmu" (i18n returnWizard.step1)
       - Fetch rental with useRental(rentalId from route params)
       - Display rental summary in AppCard: customer name, vehicle (reg + make/model), rental start/end dates, current status badge
       - Initialize return draft: updateDraft({ rentalId, step: 1 })
       - "Dalej" button (AppButton primary, bottom-fixed) -> navigate to mileage.tsx
       - Loading: LoadingSkeleton while rental loads
       - Error: if rental not found or not ACTIVE/EXTENDED, show error message and back button

    3. **return/mileage.tsx** (Step 2 - Mileage):
       - WizardStepper currentStep=2
       - Screen title: "Przebieg" (i18n returnWizard.step2)
       - Handover mileage display: AppCard showing "Przebieg przy wydaniu: {handoverMileage} km" (read from rental data, formatted with format.ts)
       - Return mileage input: AppInput with numeric keyboard, label "Przebieg przy zwrocie" (i18n returnWizard.returnMileage), suffix "km"
       - Distance driven: calculated display "Przejechane: {returnMileage - handoverMileage} km" (live calculation as user types)
       - Validation: return mileage must be >= handover mileage. Show error if less. (Use RHF with inline Zod schema: z.number().min(handoverMileage))
       - "Dalej" button -> updateDraft({ returnMileage }), navigate to checklist.tsx
       - Draft restore: if draft.returnMileage exists, pre-fill input

    4. **return/checklist.tsx** (Step 3 - Damage Checklist):
       - WizardStepper currentStep=3
       - Screen title: "Lista kontrolna" (i18n returnWizard.step3)
       - Checklist items from constants.ts (standard items): bodywork (Karoseria), interior (Wnetrze), tires (Opony), lights (Swiatla), documents (Dokumenty), cleanliness (Czystosc), fuel level (Poziom paliwa)
       - Each item as a row: label on left, Switch toggle on right. Default: off (no damage)
       - When toggled ON (damage found): expand a TextInput below for notes about the damage
       - Store in draft.checklist: Record<string, { damaged: boolean, notes: string }>
       - "Dalej" button -> updateDraft({ checklist }), navigate to notes.tsx
       - ScrollView for the list (can be longer than screen)

    5. **return/notes.tsx** (Step 4 - Notes):
       - WizardStepper currentStep=4
       - Screen title: "Uwagi" (i18n returnWizard.step4)
       - Multi-line TextInput (AppInput variant or raw TextInput with NativeWind): placeholder from i18n returnWizard.notesPlaceholder, multiline=true, numberOfLines=6, textAlignVertical="top"
       - Optional field -- can be empty
       - "Dalej" button -> updateDraft({ notes }), navigate to confirm.tsx

    6. **return/confirm.tsx** (Step 5 - Review & Confirm):
       - WizardStepper currentStep=5
       - Screen title: "Potwierdzenie zwrotu" (i18n returnWizard.step5)
       - Summary displayed in sections (AppCards):
         - Rental: customer name, vehicle
         - Mileage: handover, return, distance driven
         - Checklist: items with damage = red indicator, no damage = green indicator. Notes shown where applicable.
         - Notes: general notes text (or "Brak uwag" if empty)
       - "Zatwierdz zwrot" button (AppButton primary, bottom-fixed):
         - On press: call useReturnRental mutation with { id: rentalId, data: { returnMileage, inspectionData: checklist, notes } }
         - Loading state on button during API call
         - On success: clearDraft(), show Toast "Zwrot zarejestrowany" (i18n toasts.returnSubmitted), navigate back to rental detail (which should now show RETURNED status) or rentals list
         - On failure: Toast with errors.returnFailed, stay on screen with retry
       - ConfirmationDialog on back navigation if data entered
