    1. **src/components/SignatureScreen.tsx**: Reusable full-screen landscape signature capture per RESEARCH.md Pattern 4:
       - Props: { title: string, stepLabel: string, onConfirm: (base64Png: string) => void, onBack: () => void, loading?: boolean }
       - useEffect on mount: ScreenOrientation.lockAsync(LANDSCAPE_LEFT). Cleanup: lockAsync(PORTRAIT_UP).
       - SignatureCanvas from react-native-signature-canvas with:
         - penColor: "black", minWidth: 2, maxWidth: 3
         - webStyle hiding default footer and border
         - ref for clearSignature and readSignature
       - Header bar (48px): title text on left, stepLabel on right (e.g., "1 z 4")
       - Instruction text: i18n signatures.instruction below header
       - Canvas: flex-1 with border border-zinc-200 rounded-xl, overflow hidden
       - Bottom controls row:
         - Left: "Wyczysc podpis" (AppButton secondary, calls clearSignature)
         - Right: "Zatwierdz podpis" (AppButton primary, calls readSignature which triggers onOK -> onConfirm)
       - On confirm: Haptics.impactAsync(ImpactFeedbackStyle.Light)
       - Loading state: when loading=true, show ActivityIndicator overlay on canvas (signature being uploaded)

    2. **new-rental/signatures.tsx** (Step 5 - Signatures):
       - State: currentSignature (0-3), representing the 4 signatures in order:
         0: "Podpis klienta - umowa" (customer_page1)
         1: "Podpis pracownika - umowa" (employee_page1)
         2: "Podpis klienta - warunki" (customer_page2)
         3: "Podpis pracownika - warunki" (employee_page2)
       - Render SignatureScreen with title from i18n signatures.customerContract etc., stepLabel "{current+1} z 4"
       - On each signature confirm:
         a. Convert base64 to blob/file
         b. Call useSignContract mutation with FormData: file + signatureType + signerRole + contentHash (sha256 of contract data) + deviceInfo
         c. On success: store uploaded URL in draft.signatures array, advance to next signature
         d. On failure: retry up to 3 times with exponential backoff (1s, 2s, 4s). If all fail, show Toast with errors.signatureUploadFailed and stay on current signature with retry button.
       - After all 4 signatures captured:
         a. Create the rental: useCreateRental mutation with draft data (customerId, vehicleId, startDate, endDate, dailyRateNet)
         b. Create the contract: useCreateContract mutation linking to the rental
         c. On success: clearDraft(), navigate to success.tsx
         d. On failure: Toast with errors.rentalCreationFailed, stay on screen with retry option
       - Note: The exact order of API calls (create rental -> create contract -> sign) depends on the backend contract. Read the contracts controller to determine if contract creation requires rentalId first or if it's bundled. Adapt the flow accordingly.

    3. **new-rental/success.tsx**: Success screen per UI-SPEC:
       - Device back in portrait (SignatureScreen cleanup handles this)
       - Centered layout:
         - Checkmark icon (lucide Check in green-600 circle) at top, large (64px)

         - Body: "PDF umowy zostal wyslany na adres email klienta." (i18n wizard.successBody) — text-base text-zinc-500
       - Buttons below:
         - "Zobacz PDF" (link/text button, opens contract PDF URL via Linking.openURL — browser-based per CONTEXT.md no in-app PDF decision)
         - "Nowy wynajem" (AppButton primary) -> navigates to new-rental/index, ensures draft is cleared
         - "Powrot do pulpitu" (AppButton secondary) -> navigates to (tabs)/index
       - Prevent back navigation to signatures screen (replace rather than push, or reset stack)
