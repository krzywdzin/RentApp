# Phase 7 — UI Design Contract

> Visual and interaction contract for Photo and Damage Documentation. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Dual-surface phase:** This phase has UI on both the mobile app (Expo/React Native -- photo wizard, damage diagram) and the admin panel (Next.js -- comparison views). Design tokens are shared; component implementations differ by platform.

## Spacing Scale

Declared values (must be multiples of 4). Shared across both platforms:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, pin label padding |
| sm | 8px | Compact element spacing, thumbnail grid gap |
| md | 16px | Default element spacing, card padding, wizard step padding |
| lg | 24px | Section padding, comparison panel gutters |
| xl | 32px | Layout gaps between major content blocks |
| 2xl | 48px | Page-level vertical padding |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Photo thumbnail grid: 8px gap (sm), thumbnails at 96px square (3 per row on 320px minimum)
- Damage pin touch target: 44px minimum (accessibility, finger-tap precision)
- SVG diagram container: full-width with 16px horizontal padding
- Comparison gutter: 8px between handover and return panels
- Wizard step indicator height: 48px (progress bar + step number)
- Camera viewfinder overlay: 0px padding (full-bleed camera)

## Visual Focal Point

### Mobile -- Photo Wizard
The camera viewfinder is the primary visual anchor. It occupies the top 60% of the screen. Below it sits the position label, reference image thumbnail (small, bottom-left overlay), and the "Zrob zdjecie" CTA button centered at the bottom. The wizard step indicator runs as a horizontal progress bar at the very top (below status bar), showing 8 dots with the active one in accent color and completed ones in success green.

### Mobile -- Damage Diagram
The SVG vehicle diagram occupies the center 70% of the screen. View tabs sit above the SVG as horizontally scrollable pill buttons. The instruction text sits between the tabs and the SVG. Existing pins render as numbered circles on the diagram. The bottom area shows a scrollable list of damage pins with type and severity summary. Tapping the SVG opens the pin detail bottom sheet.

### Admin -- Comparison Page
Two-column layout (50/50 split). Left column: handover data (photos + damage diagram). Right column: return data. A tab bar at the top switches between "Zdjecia" (Photos) and "Uszkodzenia" (Damage) views. In photo mode, each row shows a position pair (handover left, return right) as 1:1 aspect ratio thumbnails. Clicking opens full-size in a dialog. In damage mode, the SVG diagram renders inline in each column with pins overlaid.

## Interaction Patterns

### Photo Wizard Flow (Mobile)

1. **Entry:** Triggered during rental activation (handover) or return process (return). Opens full-screen wizard.
2. **Step navigation:** Linear forward flow. Each step shows: position label, reference image, camera viewfinder.
3. **Capture:** Tap "Zrob zdjecie" to capture. Preview shown immediately. Options: "Dalej" (accept + next step), "Zrob ponownie" (retake).
4. **Progress:** Top progress bar fills as steps complete. Completed steps show green checkmark.
5. **Freeform:** After step 8, optional "Dodaj dodatkowe zdjecie" button appears. Each freeform photo requires a text label.
6. **Review:** After all required photos, a grid review screen shows all 8 thumbnails + any extras. Tap any thumbnail to retake.
7. **Submit:** "Zatwierdz obchod" button at bottom of review screen. Confirmation dialog before submission.
8. **Upload:** Photos upload sequentially after submission. Progress bar shows upload status per photo.

### Damage Diagram Flow (Mobile)

1. **Entry:** Automatically follows the photo wizard completion (sequential flow per CONTEXT.md).
2. **View selection:** 5 SVG views as horizontal pill tabs. Swipe gesture also navigates between views.
3. **Pin placement:** Tap anywhere on SVG. A numbered pin marker (#1, #2...) appears at tap location. Bottom sheet opens immediately.
4. **Pin detail:** Bottom sheet contains: damage type dropdown, severity selector (3 radio pills), note text input, "Dolacz zdjecie" button (opens camera), "Zapisz uszkodzenie" CTA.
5. **Pin editing:** Tap existing pin to re-open bottom sheet with prefilled data. "Usun pin" button at bottom of sheet.
6. **Return mode:** Handover pins pre-loaded as gray/locked pins (not tappable for edit). New pins appear in red with sequential numbering continuing from handover pins.
7. **No damage:** If employee adds no pins and taps "Dalej", the "no damage" confirmation sheet appears. Must explicitly confirm.
8. **Completion:** After confirming damage (or no damage), the walkthrough + damage report are submitted together.

### SVG Diagram Rendering

- **ViewBox:** All 5 SVG views use a normalized 0-1000 viewBox (coordinates stored as 0-100 percentage, rendered at 10x).
- **Car outline:** Stroke-only paths in `--muted-foreground` (zinc-400). No fill on car body.
- **Pin rendering:** Circle (r=16) with number text centered inside. Fill color = severity color. Stroke = white (2px) for contrast.
- **Pre-existing pins (return):** Fill = zinc-500, dashed stroke, number text, non-interactive.
- **New pins (return):** Fill = severity color, solid stroke, number text, tappable.
- **Tap zone:** Pins have a 44px minimum touch target (invisible enlarged hit area around the 32px visual circle).

### Comparison View (Both Platforms)

- **Photo comparison:** Swipe horizontally through 8 position pairs. Each pair shows handover (left) and return (right) thumbnails. Tap either thumbnail for full-size view. Missing photos show a placeholder with dashed border and "Brak zdjecia" text.
- **Damage comparison:** Both diagrams shown at the same SVG view simultaneously. Admin can tab between the 5 views. Handover pins in gray, return-added pins in red. Below each diagram: scrollable pin list with type, severity, and note.
- **Full-size photo viewer:** Pinch-to-zoom on mobile. Dialog with zoom controls on admin. Swipe/arrows to navigate between photos.

### Loading States

- **Wizard loading:** Skeleton pulse on camera area while permissions are acquired.
- **Photo upload:** Per-photo progress indicator (circular progress ring on each thumbnail in review grid).
- **Comparison loading:** Skeleton cards matching the 2-column photo pair layout.
- **SVG loading:** Skeleton rectangle matching diagram container dimensions.

### Toast/Feedback (Mobile)

- **Photo captured:** Brief haptic feedback (Haptics.impactAsync light). No toast.
- **Upload complete:** "Zdjecia wyslane pomyslnie" toast, 3 seconds, bottom position.
- **Upload failed:** "Blad wysylania zdjec. Ponow probe." toast, persistent until dismissed, with "Ponow" (Retry) action.
- **Walkthrough submitted:** "Obchod zatwierdzony" toast, 3 seconds.
- **Damage saved:** "Uszkodzenie zapisane" toast, 2 seconds.
