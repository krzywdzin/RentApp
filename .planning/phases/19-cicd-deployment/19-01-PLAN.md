# Plan 19-01: GitHub Actions CI/CD Pipelines

**Phase:** 19 — CI/CD & Deployment
**Requirements:** CICD-01, CICD-02, CICD-03, CICD-04

## Tasks

### Wave 1: CI Pipeline

**Task 1.1: CI workflow for PRs**
Create `.github/workflows/ci.yml`:
- Trigger: pull_request to main, push to main
- Matrix: node 20
- Steps: checkout, setup pnpm, install deps (cached), build shared, lint all, typecheck API, typecheck web, test API, test web
- Use pnpm store cache for fast installs
- Set DATABASE_URL to dummy value for API typecheck (no actual DB needed)

### Wave 2: Deploy Pipelines

**Task 2.1: API deployment workflow**
Create `.github/workflows/deploy-api.yml`:
- Trigger: push to main (paths: apps/api/**, packages/shared/**)
- Build Docker image using apps/api/Dockerfile
- Push to Railway via Railway CLI or Docker registry
- Use RAILWAY_TOKEN secret
- Run health check after deploy

**Task 2.2: Web deployment workflow**
Create `.github/workflows/deploy-web.yml`:
- Trigger: push to main (paths: apps/web/**, packages/shared/**)
- Option A: Deploy to Vercel (vercel CLI, VERCEL_TOKEN secret)
- Option B: Deploy to Railway (Docker build, RAILWAY_TOKEN)
- Include build step and verification

## Success Criteria
- [ ] CI runs on every PR
- [ ] API deploys on push to main
- [ ] Web deploys on push to main
- [ ] Workflows use pnpm caching
