SEC-07 -- S3 no defaults:
6. In `apps/api/src/storage/storage.service.ts` lines 33-37, remove the default values from S3 credential reads:
   Change:
   `accessKeyId: this.config.get<string>('S3_ACCESS_KEY', 'minioadmin'),`
   To:
   `accessKeyId: this.config.get<string>('S3_ACCESS_KEY') || 'minioadmin',`

   Wait -- this still provides a fallback. The correct approach: remove the inline fallback but keep dev working. Use the env.validation optional defaults pattern:
   - Remove the second argument ('minioadmin') from BOTH `S3_ACCESS_KEY` and `S3_SECRET_KEY` in storage.service.ts.
   - In `env.validation.ts`, add `S3_ACCESS_KEY` and `S3_SECRET_KEY` to `optionalDefaults` with `'minioadmin'` value (so dev still works via env injection, but source code has no secrets).

   This way: in dev, env.validation sets the default. In production, `S3_ACCESS_KEY` must be set in real env vars. Add `S3_ACCESS_KEY` and `S3_SECRET_KEY` to `prodRequired` array as well.
