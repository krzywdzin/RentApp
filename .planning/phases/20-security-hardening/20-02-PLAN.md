SEC-05 -- Base64 size limits:
1. In `apps/api/src/contracts/dto/sign-contract.dto.ts`:
   - Import `MaxLength` from `class-validator`
   - Add `@MaxLength(500000)` decorator on `signatureBase64` (before `@IsString()`). 500000 chars of base64 = ~375KB decoded, well above any reasonable signature PNG but prevents abuse up to the 10MB body parser limit.

2. In `apps/api/src/contracts/dto/create-contract.dto.ts`:
   - Import `MaxLength` from `class-validator`
   - Add `@MaxLength(500000)` decorator on `damageSketchBase64` (before `@IsString()`)

SEC-09 -- CSV formula injection:
1. In `apps/web/src/lib/csv-export.ts`, add formula injection sanitization. BEFORE the existing quote/delimiter escaping logic (line 13), add a check for dangerous prefixes. The standard defense is to prefix with a single quote (`'`) which Excel treats as a text qualifier:

   Inside the `.map((c) => { ... })` callback, after converting to string (`const str = ...`), add:
   ```typescript
   // Sanitize CSV formula injection
   const FORMULA_PREFIXES = ['=', '+', '-', '@', '\t', '\r'];
   const sanitized = FORMULA_PREFIXES.some(p => str.startsWith(p))
     ? `'${str}`
     : str;
   ```
   Then use `sanitized` instead of `str` in the subsequent quote/delimiter escape logic. Make sure the variable used in the `if (str.includes(';') ...)` check and the return value both use `sanitized`.

SEC-10 -- Mobile PDF signed URL:
2. In `apps/api/src/contracts/contracts.controller.ts`, add a NEW endpoint above the existing `downloadPdf` method:

   ```typescript
   @Get(':id/pdf-url')
   @Roles(UserRole.ADMIN, UserRole.EMPLOYEE)
   async getPdfUrl(@Param('id', ParseUUIDPipe) id: string) {
     const contract = await this.contractsService.findOne(id);
     if (!contract.pdfKey) {
       throw new NotFoundException('PDF not yet generated for this contract');
     }
     const url = await this.storageService.getPresignedDownloadUrl(
       contract.pdfKey,
       300, // 5-minute expiry for mobile download
     );
     return { url };
   }
   ```

3. In `apps/mobile/src/api/contracts.api.ts`, replace the `getPdfUrl` method that returns a static URL with an async method that fetches the signed URL from the API:

   Replace:
   ```typescript

   ```
   With:
   ```typescript
   getPdfUrl: async (contractId: string): Promise<string> => {
     const { data } = await apiClient.get<{ url: string }>(
       `/contracts/${contractId}/pdf-url`,
     );
     return data.url;
   },
   ```

4. IMPORTANT: Search for all callers of `contractsApi.getPdfUrl` in the mobile app (expected in `app/(tabs)/new-rental/success.tsx`) and update them to `await` the now-async call. The typical pattern will be:
