Fix three bugs:

**MBUG-03: Stale mileage restore in mileage.tsx.**
The `useEffect` that restores `draftMileage` into `mileageText` (around lines 31-36) has `mileageText` missing from its dependency array. The best fix per the audit: since `useState` is already initialized from `draftMileage` on line ~24, the `useEffect` is redundant. Remove the useEffect entirely -- the `useState` initializer already handles draft restore on mount. If `draftMileage` changes after mount (unlikely in this flow), the user's typed value should take precedence anyway.

**MBUG-04 (partial): Redundant useEffect in checklist.tsx.**
The `useEffect` with empty dependency array (around lines 35-46) that merges `draftChecklist` into local state is redundant -- the `useState` lazy initializer on lines 23-30 already does this on mount. Remove the entire `useEffect` block. The lazy initializer is the correct pattern.

**MBUG-04 (SearchBar): SearchBar local state divergence.**
In `SearchBar.tsx`, add a `useEffect` that watches the `value` prop and syncs `localValue` when the prop changes externally:
```typescript
useEffect(() => {
  setLocalValue(value);
}, [value]);
```
Place this after the existing `useState(value)` declaration (around line 12-13). This ensures that when a parent resets `value` to `''`, the SearchBar's displayed text updates accordingly.
