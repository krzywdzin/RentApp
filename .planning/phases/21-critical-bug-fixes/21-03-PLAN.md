  });

4. Add a retry mechanism: if the transaction fails due to unique constraint violation on `contractNumber` (Prisma error code P2002), retry once with an incremented count. This handles the edge case of concurrent transactions that both start with the same count.

**AREL-02: Notification message in initial create.**
In `notifications.service.ts`, find all locations where a notification is created with empty `message` then updated with the actual message (audit says lines 84-113, 138-168, 195-228, 252-277 -- four separate enqueue methods). For each:

1. Compute the message string BEFORE the `prisma.notification.create` call.
2. Include `message` in the `create` data object directly.
3. Remove the subsequent `prisma.notification.update` call that only sets the message.
4. This eliminates the window where a PENDING notification with empty message exists in the DB.

Pattern to follow for each enqueue method:
```typescript
// BEFORE (two-step):
const notification = await this.prisma.notification.create({ data: { ...fields, message: '' } });
const message = this.buildMessage(/* params */);
await this.prisma.notification.update({ where: { id: notification.id }, data: { message } });

// AFTER (single-step):
const message = this.buildMessage(/* params */);
const notification = await this.prisma.notification.create({ data: { ...fields, message } });
```

**AREL-08: SmsService lazy client initialization.**
In `sms.service.ts` (line 13), `getOrThrow('SMSAPI_TOKEN')` crashes at module init if token is missing. Fix:

1. Remove the SMSAPI client initialization from the constructor.
2. Store `configService` as a class field.
3. Add a private `getClient()` method that lazily creates the SMSAPI instance:
```typescript
private smsapi: SMSAPI | null = null;

private getClient(): SMSAPI | null {
  if (!this.smsapi) {
    const token = this.configService.get<string>('SMSAPI_TOKEN');
    if (!token) {
      this.logger.warn('SMSAPI_TOKEN not configured, SMS sending disabled');
      return null;
    }
    this.smsapi = new SMSAPI(token);
  }
  return this.smsapi;
}
```
4. In the `send()` method, call `const client = this.getClient(); if (!client) return;` to skip sending when token is unavailable.
5. Add a `private readonly logger = new Logger(SmsService.name)` for the warning log.
