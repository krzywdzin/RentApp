**AREL-05: Photo upload -- DB record before S3, with cleanup on failure.**
In `photos.service.ts` (around lines 95-111), files are uploaded to S3 before the DB record is created. Fix the ordering:

1. Create the DB record FIRST (with the S3 key that will be used):
```typescript
const key = `walkthroughs/${walkthroughId}/${position}/${uuid}`;
const record = await this.prisma.walkthroughPhoto.create({
  data: { walkthroughId, position, photoKey: key, thumbnailKey: thumbKey, ... }
});
```
2. THEN upload to S3:
```typescript
try {
  await this.storage.upload(key, photoBuffer);
  await this.storage.upload(thumbKey, thumbnailBuffer);
} catch (uploadError) {
  // Clean up the DB record if S3 upload fails
  await this.prisma.walkthroughPhoto.delete({ where: { id: record.id } });
  throw uploadError;
}
```
3. This ensures no orphaned S3 objects. If the DB create fails, no S3 upload happens. If S3 upload fails, the DB record is cleaned up.

**AREL-06: replacePhoto -- upload new before deleting old.**
In `photos.service.ts` (around lines 252-284), old S3 objects are deleted BEFORE new ones are uploaded. Reverse the order:

1. Upload new files to S3 first (using new keys).
2. Update the DB record with the new keys.
3. THEN delete the old S3 objects.
4. If the new upload fails, the old files are still intact (no data loss).

Restructure to:
```typescript
// 1. Upload new files
const newKey = generateNewKey(...);
await this.storage.upload(newKey, newPhotoBuffer);
await this.storage.upload(newThumbKey, newThumbnailBuffer);

// 2. Update DB record
const oldPhotoKey = existingPhoto.photoKey;
const oldThumbKey = existingPhoto.thumbnailKey;
await this.prisma.walkthroughPhoto.update({
  where: { id: existingPhoto.id },
  data: { photoKey: newKey, thumbnailKey: newThumbKey }
});

// 3. Clean up old files (best-effort -- if this fails, old files are orphaned but no data loss)
try {
  await this.storage.delete(oldPhotoKey);
  await this.storage.delete(oldThumbKey);
} catch (cleanupError) {
  this.logger.warn(`Failed to delete old photo files: ${cleanupError.message}`);
}
```
