       - In `alert-scanner.service.ts`, the `getWarsawDateRange` method currently uses `setHours(0,0,0,0)` which sets hours in the system's local timezone (UTC on Railway/Docker), not Warsaw time.
       - Replace with timezone-aware calculation. Check if `date-fns-tz` is already installed (`grep date-fns-tz apps/api/package.json`). If yes, use it. If not, use manual UTC offset calculation.
       - **With date-fns-tz approach:**
         ```typescript
         import { startOfDay, endOfDay } from 'date-fns';
         import { toZonedTime, fromZonedTime } from 'date-fns-tz';

         const TIMEZONE = 'Europe/Warsaw';
         // Get current time in Warsaw
         const nowWarsaw = toZonedTime(new Date(), TIMEZONE);
         // Add daysFromNow
         const targetWarsaw = new Date(nowWarsaw);
         targetWarsaw.setDate(targetWarsaw.getDate() + daysFromNow);
         // Get start/end of day in Warsaw, converted back to UTC
         const startUtc = fromZonedTime(startOfDay(targetWarsaw), TIMEZONE);
         const endUtc = fromZonedTime(endOfDay(targetWarsaw), TIMEZONE);
         ```
       - **Without date-fns-tz (manual approach):**
         ```typescript
         // Get Warsaw date string
         const warsawNow = new Date().toLocaleDateString('en-CA', { timeZone: 'Europe/Warsaw' });
         // Parse and add days
         const target = new Date(warsawNow);
         target.setDate(target.getDate() + daysFromNow);
         const targetStr = target.toISOString().split('T')[0];
         // Create UTC boundaries for that Warsaw date
         // Warsaw is UTC+1 (CET) or UTC+2 (CEST)
         // Use Intl to get the offset dynamically
         const formatter = new Intl.DateTimeFormat('en-US', { timeZone: 'Europe/Warsaw', timeZoneName: 'shortOffset' });
         // Simpler: just construct dates using the timezone
         const startUtc = new Date(`${targetStr}T00:00:00+01:00`); // approximate
         ```
       - Prefer the date-fns-tz approach if available -- it handles CET/CEST transitions correctly.
       - If date-fns-tz is NOT installed, install it: `cd apps/api && npm install date-fns-tz`
